# Troubleshooting

## Server Does Not Start

Use a different port:

```bash
PORT=3100 npm start
```

## Browser Loads Blank Page

- Check the browser console for module import errors.
- Run `npm install` so `/node_modules/fflate/esm/browser.js`,
  `/node_modules/kicad-toolkit/`, and
  `/node_modules/altium-toolkit/` exist for the import map.
- Verify `src/main.mjs` matches the script tag in `src/index.html`.

## Board Does Not Render

- Confirm the selected file is a supported KiCad `.kicad_pcb`, Altium
  `.PcbDoc`, KiCad project ZIP, Altium ZIP containing `.PcbDoc`, or PCB
  Styler project ZIP.
- Confirm the KiCad board has `Edge.Cuts` geometry; without it, the renderer falls back to content bounds.
- Confirm Altium input is a PCB document (`.PcbDoc`), not a schematic
  document (`.SchDoc`).
- Very large silk polygons can take longer to render because the SVG preserves their point data.

## PNG Looks Cropped

PNG export uses the current SVG viewBox. If the KiCad file has missing or unusual outline geometry, check whether `Edge.Cuts` contains a valid closed outline.

## Project ZIP Does Not Restore Settings

- Confirm the archive contains `settings.json` generated by PCB Styler.
- Confirm the archive also contains the referenced `.kicad_pcb` or `.PcbDoc` file.
- Opening a normal KiCad project ZIP still loads the board but has no PCB Styler settings to restore.
- Opening a normal Altium ZIP that contains only a `.PcbDoc` still loads the
  board but has no PCB Styler settings to restore.

## LIVE Works Locally But Not After Deployment

- Run `npm run build:static` and verify `.deploy-src/index.html` contains `/main.mjs?v=<package version>`.
- Confirm the FTP workflow uploaded `.deploy-src/` to the LIVE document root instead of raw `src/`.
- Confirm production `node_modules/` was uploaded after dependency changes so `/node_modules/kicad-toolkit/`, `/node_modules/altium-toolkit/`, and `/node_modules/fflate/` resolve.
- Confirm `api/app-meta.php` and root `package.json` are deployed if the footer version is blank.

## Tests Fail After Refactor

- Ensure moved files are reflected in test imports.
- Keep line-limit and structure tests updated with intentional layout changes.
